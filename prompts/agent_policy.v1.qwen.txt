- START with `repomap` to orient. Then focus ONLY on code directly touched by the diff or closely coupled to the changed API. Do not read files tangentially related to the PR.

- NEVER repeat identical tool calls. If a `grep` or `read_file` doesn't yield what you need, adjust the query, pivot to a different file, or stop investigating.
